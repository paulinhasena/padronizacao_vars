# Nomes e padrão do projeto

## Objetivo

Deixar o projeto mais natural para apresentação interna, usando português onde faz sentido e inglês apenas onde é padrão técnico.

## Mantido em inglês

- backend
- frontend
- src
- components
- package.json
- requirements.txt
- App.jsx
- api.js

Motivo: são nomes comuns de mercado e ferramentas esperam essa estrutura.

## Alterado para português

- INICIAR_PADRONIZADOR.bat
- GERAR_FRONTEND_PESSOAL.bat
- LEIA_ME_PRIMEIRO.txt
- GUIA_DESENVOLVEDORA.md

Motivo: são arquivos lidos/usados por pessoas, então devem ser claros.
