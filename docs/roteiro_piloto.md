# Roteiro de Piloto — Data Naming AI

## Objetivo do piloto

Validar se o MVP reduz esforço manual na padronização de variáveis e melhora consistência de nomes lógicos/físicos.

## Público recomendado

Começar com 1 a 3 pessoas próximas:

- alguém técnico;
- alguém que sofre com nomenclatura/modelagem;
- alguém que possa dar feedback sincero.

## Como conduzir

1. Explique o problema.
2. Mostre o fluxo em 3 minutos.
3. Peça para a pessoa testar com 5 a 10 variáveis.
4. Colete feedback.
5. Ajuste o dicionário/regras.

## Perguntas de feedback

- O fluxo ficou fácil de usar?
- A saída ajudaria na rotina?
- O nome físico fez sentido?
- O nome lógico ficou claro?
- A descrição ficou útil?
- Qual formato de saída seria mais útil?
- O que faltou para isso virar uma ferramenta realmente útil?

## Métrica simples

Antes:
- quanto tempo levaria para padronizar X variáveis manualmente?

Depois:
- quanto tempo levou usando o MVP?
- quantas sugestões precisaram de ajuste?
- quais termos faltaram no dicionário?

## Frase para apresentar

"Estou validando um MVP local, sem custo de infraestrutura, para padronização de variáveis com base em dicionário de termos e regras de nomenclatura. A ideia é provar valor antes de discutir escala ou publicação corporativa."
